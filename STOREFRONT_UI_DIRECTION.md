# Storefront UI Direction: Saree Brand Experience

## Purpose
Define a clear visual and interaction direction for the customer-facing storefront.

Admin UI can remain simple and utilitarian.
Storefront must feel elegant, colorful, pleasant, and conversion-focused.

## Brand Personality
1. Elegant
2. Warm
3. Festive
4. Contemporary-Indian
5. Trustworthy

## Visual Principles
1. Rich but not cluttered
2. High visual hierarchy (hero, collection, product cards)
3. Fabric-first storytelling (texture, drape, weave, color)
4. Soft luxury feel with clear readability
5. Mobile-first polish with premium desktop refinement

## Color Strategy
Use layered palettes with strong accents and calm neutrals.

### Base Neutrals
1. Ivory / off-white backgrounds
2. Soft beige sections
3. Charcoal for body text

### Accent Families (seasonal or collection-based)
1. Ruby / deep red
2. Marigold / saffron
3. Emerald / leaf green
4. Royal blue
5. Rose / blush accents

### Usage Rules
1. One dominant accent per page section
2. Avoid too many competing saturated colors in the same viewport
3. Keep text contrast WCAG-friendly

## Typography Direction
1. Headings: elegant serif with modern proportions
2. Body/UI text: clean sans-serif for readability
3. Price and CTA emphasis with strong weight contrast
4. Generous line-height and breathing room

## Layout & Components

### Homepage
1. Hero banner with collection storytelling
2. Featured categories (wedding, festive, daily wear, silk, cotton)
3. New arrivals strip
4. Bestsellers grid
5. Occasion-led recommendation blocks
6. Customer trust row (quality, easy returns, shipping clarity)

### Product Listing
1. Sticky filter + sort bar
2. Grid with card image focus
3. Quick color swatch preview
4. Price, offer tags, and stock hints

### Product Detail Page
1. Large image gallery with zoom
2. Color variants as swatches
3. Size/blouse info area
4. Delivery estimate block
5. Add to cart + buy now CTAs
6. Related products carousel

## Motion & Interaction
1. Gentle fade/slide reveals for sections
2. Smooth hover lift on product cards
3. Swatch hover updates preview image
4. Subtle micro-feedback on add-to-cart actions
5. Keep animation fast and non-distracting

## Imagery Direction
1. Bright, high-quality saree visuals
2. Consistent framing and lighting
3. Mix of model shots + texture close-ups
4. Occasion context (wedding, festive, office, casual)
5. Avoid cluttered backgrounds that hide fabric detail

## Trust & Conversion UX
1. Show return policy and delivery info near CTAs
2. Clear stock availability messaging
3. Transparent pricing (MRP, offer, final)
4. Crisp checkout progression indicators
5. Minimize friction on mobile checkout

## Responsive Behavior
1. Mobile: single-column product details, sticky bottom CTA
2. Tablet: 2-column product grid
3. Desktop: 3-4 column grid with persistent filters

## Accessibility & Readability
1. Maintain strong text/background contrast
2. Keyboard-focus visible on interactive elements
3. Minimum 16px body text equivalent on mobile
4. Touch targets sized for thumbs

## What to Avoid
1. Copying competitor layouts 1:1
2. Overusing gradients, shadows, or animation
3. Excessive text blocks before product discovery
4. Dark-heavy themes that mute saree colors

## Implementation Guidance
1. Build a storefront design token file first (color, type, spacing, radius, shadows)
2. Create reusable storefront components:
   - product card
   - swatch selector
   - price block
   - trust badges
   - filters drawer
3. Keep admin and storefront component themes separate

## Success Checklist
1. First impression feels premium and culturally relevant
2. Product discovery is fast and enjoyable
3. Saree colors and fabric details stand out
4. Mobile shopping experience feels smooth and modern
5. UI is distinct, not derivative
